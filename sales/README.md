# Sales

The knowledge layer: who we sell to, how we sell, and the processes for Eik & Friends' core
business lines. Prompts and agents reference these playbooks so messaging stays consistent and
grounded in strategy rather than improvised each time.

## Contents
| File | What it covers |
| --- | --- |
| [`icp.md`](icp.md) | Ideal Customer Profile — who is (and isn't) worth pursuing. |
| [`methodology.md`](methodology.md) | Our end-to-end sales method and stage definitions. |
| [`playbook-corporate-events.md`](playbook-corporate-events.md) | Selling corporate event bookings. |
| [`playbook-restaurant-partnerships.md`](playbook-restaurant-partnerships.md) | Building ongoing venue/restaurant partnerships. |

## How to use this module
- Writing outreach? Check [`icp.md`](icp.md) for fit and [`methodology.md`](methodology.md) for the
  current stage's goal.
- Working a specific business line? Open its playbook for the motion, common objections, and value
  points.
- These documents evolve as we learn — update them when a pattern proves itself (per
  [`docs/PRINCIPLES.md`](../docs/PRINCIPLES.md)).

## Note on data
These are **strategy and process** documents. Live customer and deal data lives in the CRM
(Notion / Google Sheets), modeled in [`crm/`](../crm/) — not here.
